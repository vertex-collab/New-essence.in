🤖 AI Style Assistant
✦ AI Agent

Power the floating chat assistant on your storefront. Customers can ask for outfit advice, sizing help, order support and more — all answered by AI in real time.

⚠️ OpenAI has CORS restrictions — direct browser calls are blocked by OpenAI's security policy. This site will attempt to route through a CORS proxy (corsproxy.io), but it may not always work. For a reliable setup, Anthropic (Claude) or Google Gemini are recommended as they support browser-direct calls.
Leave blank to use the default prompt. You can add info about your store, policies, or personality.
How to get your API key:
🟠 Anthropic (Claude): console.anthropic.com → API Keys
🟢 OpenAI: platform.openai.com → API Keys
🔵 Google Gemini: aistudio.google.com → Get API Key
Your key is stored in localStorage and never sent anywhere except directly to the AI provider.
📧 Email Notifications
How to set up (free, 200 emails/month):
1. Go to emailjs.com → Sign up free
2. Add Email Service → connect your Gmail
3. Create Email Template → use these variables:
{{order_id}} {{customer}} {{email}} {{items}} {{total}} {{address}} {{payment}} {{status}} {{message}}
✓ Emails are sent to BOTH the store owner AND the customer automatically on every order.
4. Copy your Service ID, Template ID, Public Key
5. Paste above → Save → Send Test Email ✓

⚡ Razorpay Integration
○ Enter your Razorpay Key ID to enable instant UPI/card payments.
How to get Razorpay Key ID:
1. Go to razorpay.com → Sign up free
2. Dashboard → Settings → API Keys
3. Generate Key → Copy Key ID (starts with rzp_test_ or rzp_live_)
4. Use rzp_test_ for testing, rzp_live_ for real payments
5. Once saved, Razorpay button appears in UPI section for customers
🗄️ Supabase Database
How to get your keys:
1. Go to supabase.com → your project
2. Click Settings → API
3. Copy Project URL and anon public key
4. Paste above and click Save & Connect
📋 Database Setup SQL

Run this SQL in your Supabase → SQL Editor once to create all tables:

create table products ( id bigint primary key, name text, price numeric, original numeric, category text, stock int, emoji text, image text, description text, badge text, colors jsonb, sizes jsonb ); create table orders ( id text primary key, customer text, email text, address text, items jsonb, total numeric, discount text, payment text default 'card', payment_txn_id text, payment_verified boolean default false, status text default 'pending', date date default now() ); create table discount_codes ( code text primary key, type text, value numeric, min_order numeric default 0, uses int default 0, max_uses int default 0, expiry date, active boolean default true ); -- Allow public read/write (for demo): alter table products enable row level security; alter table orders enable row level security; alter table discount_codes enable row level security; create policy "public" on products for all using (true) with check (true); create policy "public" on orders for all using (true) with check (true); create policy "public" on discount_codes for all using (true) with check (true);
